## ADDED Requirements

### Requirement: 所有 HTTP 响应声明 UTF-8 字符集
生产环境的 Nginx SHALL 为所有响应（接口代理与静态资源）声明 UTF-8 字符集，确保中文内容在任何客户端默认编码（含中文 Windows 的 GBK）下都不出现乱码。

#### Scenario: API JSON 错误响应为 UTF-8
- **WHEN** 客户端请求任意 `/api/` 接口并触发后端返回含中文的错误响应
- **THEN** 响应的 `Content-Type` 声明 UTF-8 字符集，中文文本在浏览器中正确显示、无乱码

#### Scenario: 静态资源为 UTF-8
- **WHEN** 浏览器加载前端静态资源（HTML/CSS/JS）
- **THEN** 资源以 UTF-8 提供，页面所有中文（含登录页）正确显示、无乱码

### Requirement: 登录页无效手机号提示正确显示中文
登录页在手机号无效时给出的提示 MUST 以正确的中文显示，不得出现乱码。

#### Scenario: 登录页手机号格式错误提示无乱码
- **WHEN** 用户在登录页输入无效手机号（如不足 11 位、非手机号段）并尝试登录
- **THEN** 提示文字以正确中文显示（无乱码），无论该提示来自前端校验还是后端回显

#### Scenario: 前端拦截非法手机号前缀
- **WHEN** 用户输入 11 位但非合法手机号段（如以 0/1 开头的不合规号段）的号码
- **THEN** 前端在提交前即给出中文格式错误提示，无需依赖后端回显
