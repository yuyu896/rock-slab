## Context

当前 PC 端用户面板（`MainLayout.vue`）通过居中弹窗方式展示，触发源是侧边栏底部用户卡片。面板内头像、个人信息、操作按钮纵向堆叠，间距紧凑。头像仅支持文件上传，无预设选项。

后端 User 模型已有 `avatar`（ImageField）字段，支持上传和删除。前端通过 `uploadAvatar` / `deleteAvatar` API 操作头像。

技术栈：Vue 3 + TypeScript + Element Plus（CSS 自定义属性体系），Django REST Framework 后端。

## Goals / Non-Goals

**Goals:**
- 将面板从页面居中弹窗改为锚定在侧边栏底部用户卡片附近弹出（类似 Popover）
- 优化面板内部布局，建立清晰的信息层级
- 新增系统预设头像（8-10 个简约 SVG 几何头像），支持快速选择
- 预设头像与自定义上传互斥，选择预设时自动清除自定义头像，反之亦然
- 保持移动端不受影响（移动端使用独立 Profile 页面）

**Non-Goals:**
- 不重构移动端 Profile 页面
- 不引入第三方头像生成服务
- 不做头像裁剪/编辑功能
- 不修改后端文件存储方式（仍使用 Django ImageField + 本地/Media 存储）

## Decisions

### 1. 面板定位：侧边栏锚定 Popover 替代居中 Modal

**选择**：使用绝对定位 Popover，锚定在侧边栏 `.sidebar-footer` 元素旁侧
**理由**：用户点击侧边栏底部卡片后，面板从点击处弹出更符合空间认知。居中弹窗切断操作上下文。
**备选**：Drawer（右侧抽屉）— 过重，且与侧边栏交互距离过远。

实现方式：
- 将 `.user-panel` 从 `position: fixed; inset: 0` 的遮罩层改为 `position: absolute`，相对于 `.sidebar-footer` 定位
- 面板固定在侧边栏右侧，底部对齐用户卡片
- 保留半透明遮罩，点击遮罩关闭面板
- 面板宽度 380px，高度自适应（max-height: 80vh）

### 2. 面板内部布局重构

**选择**：采用左右分栏或紧凑型卡片式布局
**理由**：当前全部纵向堆叠导致面板过长。新布局将头像/用户信息作为顶部卡片区域，编辑和操作放在下方。

新布局结构：
```
┌─────────────────────────────┐
│  ← 返回/标题          ✕ 关闭 │  ← header
├─────────────────────────────┤
│      [  头像 80px  ]         │  ← 顶部用户卡片
│      张三 · 行政经理         │     (仅展示，不可点击)
│      北京分公司              │
├─────────────────────────────┤
│  头像管理                    │  ← 可折叠的头像选择区
│  ○ ○ ○ ○ ○  [自定义上传]    │     预设头像网格 + 上传按钮
├─────────────────────────────┤
│  姓名  [____张三____]       │  ← 信息编辑区
│  手机  138****8000           │
│  分公司  北京分公司           │
│        [ 保存修改 ]          │
├─────────────────────────────┤
│  🔒 修改密码            >    │  ← 操作区
│  🚪 退出登录                 │
└─────────────────────────────┘
```

### 3. 预设头像方案：内联 SVG 几何图形

**选择**：使用代码内联的 SVG 几何头像（非图片文件）
**理由**：
- 无需额外静态资源管理，不增加构建体积
- 与系统主题色自然匹配，可响应主题切换
- 简约风格，符合企业内部系统定位

**备选**：PNG/SVG 静态文件 — 增加资源管理复杂度，不如代码内联灵活。

预设头像设计：10 个简约几何图案头像，使用不同配色方案：
- 抽象几何（圆形、三角形、方形组合）
- 每个头像由 2-3 种颜色组成
- 颜色从主题色系派生（primary, teal, amber, rose, indigo, emerald, orange, cyan, violet, slate）

### 4. 头像存储：system_avatar 字段

**选择**：User 模型新增 `system_avatar`（CharField，可为空，存储预设头像标识）
**理由**：
- 预设头像由前端 SVG 渲染，后端只需存储标识符
- `avatar`（自定义上传）和 `system_avatar`（预设）互斥
- 前端优先级：`avatar`（自定义）> `system_avatar`（预设）> 姓名首字母

**备选**：使用整数枚举 — CharField 更灵活，预设头像标识可直接映射到前端组件名。

### 5. 面板定位技术实现

面板定位使用 CSS `position: absolute` + JS 动态计算：
- `.sidebar-footer` 设置 `position: relative` 作为定位锚点
- 面板使用 `position: absolute; left: 100%; bottom: 0` 定位在侧边栏右侧
- 面板与侧边栏之间留 8px 间距
- 考虑视口底部溢出：面板 max-height 限制，超出时上移

## Risks / Trade-offs

- **面板溢出风险**：侧边栏在底部时，面板可能超出视口底部 → 设置 max-height + overflow-y: auto，并使用 JS 检测是否需要上移
- **预设头像个性化不足**：几何图案可能无法满足所有用户偏好 → 保留自定义上传作为补充
- **头像优先级逻辑**：`avatar` 和 `system_avatar` 互斥需要前后端一致 → 前端选择预设时调用删除自定义头像，上传自定义时清除 system_avatar 字段
