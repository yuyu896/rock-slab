## Context

组织架构页面（`Organization.vue`）已有三个标签页：组织架构（人员汇报树）、区域管理（卡片网格）、分公司管理（表格）。人员管理功能需要在同一页面新增第四个标签页，复用已有的 `@/api/users` 模块和后端 `UserViewSet`（`/api/users/`），不新增后端接口。

## Goals / Non-Goals

**Goals:**
- 在组织架构页面新增"人员管理"标签页，提供表格视图展示全部人员
- 支持按姓名/手机号关键词搜索
- 支持按角色、所属区域、所属分公司、状态进行筛选
- 提供新增人员（开通账号）的完整表单
- 支持编辑人员基本信息
- 支持启用/停用账号状态切换
- 支持删除人员（二次确认）

**Non-Goals:**
- 不新增后端 API，完全复用现有 `UserViewSet`
- 不实现批量操作（批量导入、批量删除等）
- 不实现人员详情独立页面
- 不实现密码重置功能（已有独立的修改密码接口）

## Decisions

1. **复用现有 API 而非新建后端接口**：`UserViewSet` 已提供完整的 CRUD（list/create/update/destroy）和过滤（role/branch/keyword/status），前端直接使用即可，减少后端改动。

2. **标签页模式而非独立页面**：人员管理与区域管理、分公司管理并列在同一页面，通过标签页切换，保持组织架构模块的内聚性。

3. **表格视图而非卡片视图**：人员数据字段较多且结构化，表格更适合快速浏览和对比。

4. **搜索和筛选放在表格上方**：采用搜索框 + 下拉筛选器的组合布局，筛选后即时刷新数据。

## Risks / Trade-offs

- 后端 `UserViewSet` 当前 `pagination_class = None`，人员数量增长后可能需要重新引入分页
- 筛选参数 `branch` / `region` 需与后端 `UserFilterSet` 的字段名匹配
