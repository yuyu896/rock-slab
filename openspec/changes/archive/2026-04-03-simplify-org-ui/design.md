## Context

当前 `Organization.vue` 包含四个标签页（orgchart、regions、branches、personnel），其中 personnel（人员管理）与 orgchart 中的人员详情面板功能重复。侧边栏顶部有"折叠架构"和"展开架构"按钮，但用户可以直接点击树节点图标进行展开/收起，功能冗余。

## Goals / Non-Goals

**Goals:**
- 简化组织架构模块 UI，减少冗余
- 将编辑、删除操作放在节点详情面板中，操作更直观
- 删除不必要的折叠/展开全部按钮

**Non-Goals:**
- 不修改后端 API
- 不改变组织架构树的数据结构
- 不修改人员数据表格的列定义

## Decisions

1. **删除 personnel 标签页**
   - 理由：与 orgchart 中的人员列表重复，用户可在 orgchart 中查看和管理所有人员
   - 替代方案：保留但合并到 orgchart → 增加复杂度，不采用

2. **在节点详情面板添加操作按钮**
   - 理由：用户选中节点后，在右侧面板直接操作更符合直觉
   - 按钮位置：详情面板顶部，节点名称旁边
   - 按钮样式：使用现有的 btn-edit / btn-delete 样式类

3. **删除折叠/展开全部按钮**
   - 理由：树节点自带展开/收起图标，用户可逐个操作；批量操作使用场景少
   - 替代方案：保留但移到右键菜单 → 增加实现复杂度，不采用

## Risks / Trade-offs

- **风险**: 用户习惯在 personnel 标签页管理人员 → **缓解**: 在 orgchart 中提供同等功能（编辑、删除按钮）
- **风险**: 删除折叠全部后，大树展开状态管理变繁琐 → **缓解**: 默认展开第一级，用户按需展开
