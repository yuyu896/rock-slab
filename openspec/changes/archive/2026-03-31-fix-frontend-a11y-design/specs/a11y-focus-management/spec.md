# 可访问性：焦点管理

所有交互元素必须支持键盘焦点导航，焦点状态必须清晰可见。

## ADDED Requirements

### Requirement: 交互元素必须显示焦点样式

所有可交互元素（按钮、链接、输入框、可选择项）必须显示清晰的焦点指示器。

#### Scenario: 键盘焦点可见
- **WHEN** 用户使用 Tab 键导航到交互元素
- **THEN** 元素必须显示清晰的视觉焦点指示器
- **AND** 焦点指示器对比度至少 3:1
- **AND** 焦点指示器不遮挡元素内容

#### Scenario: 鼠标点击不显示焦点环
- **WHEN** 用户使用鼠标点击按钮或链接
- **THEN** 不应显示焦点环（使用 `:focus-visible` 区分）
- **AND** 元素仍保持焦点状态（可响应键盘事件）

#### Scenario: 焦点顺序遵循视觉顺序
- **WHEN** 用户使用 Tab 键导航
- **THEN** 焦点顺序必须遵循从左到右、从上到下的视觉顺序
- **AND** 不应跳过可见的交互元素

### Requirement: 模态框必须实现焦点陷阱

模态框打开时，焦点必须被限制在模态框内部，用户无法将焦点移到模态框外部。

#### Scenario: 模态框焦点陷阱
- **WHEN** 模态框打开
- **THEN** 焦点自动移动到模态框内第一个可聚焦元素
- **AND** 按 Tab 键焦点仅在模态框内循环
- **AND** 按 Shift+Tab 焦点反向循环但仍在模态框内

#### Scenario: Escape 键关闭模态框
- **WHEN** 模态框打开
- **AND** 用户按下 Escape 键
- **THEN** 模态框关闭
- **AND** 焦点返回到打开模态框的触发元素

#### Scenario: 模态框关闭后焦点恢复
- **WHEN** 模态框以任何方式关闭
- **THEN** 焦点必须返回到打开模态框的元素
- **AND** 如果触发元素不存在，焦点返回到 `document.body`

### Requirement: 焦点样式必须尊重减少动画偏好

用户设置 `prefers-reduced-motion: reduce` 时，焦点过渡动画应当简化或禁用。

#### Scenario: 减少动画模式下的焦点样式
- **WHEN** 用户系统设置 `prefers-reduced-motion: reduce`
- **THEN** 焦点样式过渡应立即生效（无动画）
- **AND** 焦点指示器仍然可见
