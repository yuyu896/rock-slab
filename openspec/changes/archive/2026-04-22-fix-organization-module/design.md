## Context

组织架构模块包含 4 个管理标签页：区域管理、分公司管理、行政组、人员管理。后端 User 的"删除"操作实际是软删除（`perform_destroy` 将 `status` 改为 `inactive`），但前端人员列表无任何过滤，导致"删除"后人员仍可见。

组织架构树（左侧 sidebar）已正确过滤 `status === 'active'` 的用户，但人员管理表格没有。

## Goals / Non-Goals

**Goals:**
- 人员管理列表默认只显示 active 用户
- "删除"按钮改为"停用"，消除误导
- 验证组织架构模块全部 CRUD 功能正常

**Non-Goals:**
- 不改后端软删除机制（软删除是正确的设计）
- 不新增"彻底删除"功能
- 不改区域/分公司/行政组的现有逻辑（除非验证发现 bug）

## Decisions

### 1. 前端过滤而非后端过滤

在 `fetchUsers` 返回数据后，前端默认过滤掉 `inactive` 用户。后端 API 保持返回全部用户，通过状态筛选切换可见性。

**理由**: PersonnelManager 已有 `personnelStatusFilter` 筛选 UI，只需调整默认值即可。

### 2. 按钮和确认文案统一改为"停用"

**理由**: 后端实际行为是停用而非删除，文案应匹配实际语义。

## Risks / Trade-offs

- **[已停用用户不可见]** → 用户可能找不到被停用的人。通过状态筛选下拉框解决，选项包含"全部/在职/已停用"。
- **[其他模块的 bug]** → 验证过程中可能发现新问题，需要单独处理。
