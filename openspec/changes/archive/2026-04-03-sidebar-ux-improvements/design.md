# 侧边栏 UX 改进设计

## 1. 用户信息区域重构

### 当前设计
```
┌─────────────────────────┐
│ [头像] 用户名           │
│        角色    [退出]   │
└─────────────────────────┘
```
- 头像可点击更换
- 退出按钮独立显示
- 无个人信息编辑入口

### 新设计
```
┌─────────────────────────┐
│ [头像] 用户名     [▼]   │  ← 点击展开下拉菜单
└─────────────────────────┘

下拉菜单:
┌─────────────────────────┐
│ 📷 修改头像             │
│ 👤 修改个人信息          │
│ ─────────────────────── │
│ 🚪 退出登录             │
└─────────────────────────┘
```

### 交互说明
- 整个用户信息区域可点击
- 点击后弹出下拉菜单
- 菜单选项：
  - **修改头像**：打开头像上传对话框
  - **修改个人信息**：跳转到个人信息编辑页面
  - **退出登录**：确认后退出系统

## 2. 视觉清晰度优化

### 当前问题
- 文字颜色对比度不足，`--color-text-secondary` 和 `--color-text-tertiary` 在侧边栏背景上辨识度低

### 优化方案
- 提升 `--color-text-primary` 对比度至 WCAG AA 标准（至少 4.5:1）
- 菜单文字使用 `--color-text-primary` 替代 `--color-text-secondary`
- 角色标签保持使用 `--color-text-tertiary`，但提升至最小对比度 3:1

### CSS 变量调整
```css
/* 侧边栏专用文字颜色 */
--sidebar-text-primary: rgba(0, 0, 0, 0.85);    /* 主要文字 */
--sidebar-text-secondary: rgba(0, 0, 0, 0.65);  /* 次要文字 */
--sidebar-text-tertiary: rgba(0, 0, 0, 0.45);   /* 辅助文字 */
```

## 3. 菜单顺序调整

### 当前顺序
1. 工作台
2. 资产分类
3. 资产管理
4. 资产盘点
5. 组织架构
6. 统计报表 ← 位置过低

### 新顺序
1. 工作台
2. **统计报表** ← 移动到工作台下方
3. 资产分类
4. 资产管理
5. 资产盘点
6. 组织架构

### 理由
- 工作台和统计报表是高频访问功能
- 将核心功能放在视觉最容易触及的位置（上半部分）
