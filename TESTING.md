## 建议测试路径

### 路径一：管理员视角（全面体验）
1. 使用管理员账号登录（凭据见服务器）
2. 查看工作台/仪表盘
3. 浏览资产管理 → 查看资产列表
4. 浏览分类管理 → 查看资产分类
5. 浏览组织管理 → 查看组织架构
6. 尝试创建一笔采购单
7. 查看盘点任务
8. 查看报表统计

### 路径二：普通专员视角（日常操作）
1. 使用专员账号登录（凭据见服务器）
2. 查看工作台
3. 搜索/查看自己名下的资产
4. 查看通知消息

### 路径三：手机端体验
1. 在手机浏览器中打开 https://qhpanpan.top
2. 使用任意测试账号登录
3. 体验移动端界面（扫码、审批等）

## 测试要点

### 功能测试
- [ ] 登录/登出是否正常
- [ ] 各页面能否正常加载
- [ ] 数据列表、筛选、搜索是否正常
- [ ] 表单创建、编辑是否正常提交
- [ ] 数据导入/导出是否正常

### 兼容性测试
- [ ] 电脑端 Chrome / Edge
- [ ] 手机端浏览器（微信内置浏览器 / Safari）
- [ ] 不同屏幕尺寸下布局是否正常

### 体验反馈
- [ ] 页面加载速度如何
- [ ] 操作流程是否直观
- [ ] 是否有不符合预期的行为
- [ ] 是否有文字/排版错误

## 自动化测试

### 后端测试 (pytest)

```bash
cd backend
pip install -r requirements.txt
pytest                        # 运行全部测试
pytest --cov=apps             # 带覆盖率报告
pytest tests/test_auth.py     # 运行单个文件
```

| 测试文件 | 覆盖范围 |
|---|---|
| `tests/test_auth.py` | 登录/登出、Token 过期、密码修改、限流、账号锁定 |
| `tests/test_permissions.py` | 五级角色权限、数据范围隔离 |
| `tests/test_transfers.py` | 采购入库、领用出库、审批流程 |
| `tests/test_inventories.py` | 盘点任务状态流转、盘点执行、审批 |

### 前端测试 (vitest)

```bash
cd frontend
npm install
npm run test                  # 运行全部测试
npm run test:coverage         # 带覆盖率报告
```

| 测试文件 | 覆盖范围 |
|---|---|
| `tests/store/user.test.ts` | 登录、登出、fetchProfile、角色判断 |
| `tests/store/asset.test.ts` | fetchAssets、筛选、分页、错误处理 |
| `tests/hooks/usePermission.test.ts` | 角色层级、权限判断 |
